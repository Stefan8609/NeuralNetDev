# Docs Manifest

## Intent

The `docs/` directory captures the design decisions that should guide implementation.
Read here before making structural changes.

## Files

- `DEVELOPMENT.md`: branch, commit, and repository working norms.
- `ARCHITECTURE.md`: package boundaries and intended seismic ML workflow.
- `ROADMAP.md`: milestone plan from scaffold to usable baseline models.

## Editing guidance

- Update `ARCHITECTURE.md` when adding or reshaping package modules.
- Update `ROADMAP.md` when a milestone is completed or re-scoped.
- Keep docs aligned with code; if a module exists, it should be represented here.
